　# 読書記録アプリ 仕様書

## アプリ概要

本を検索し、「気になる本」を保存・整理する読書記録アプリ。

単なる蔵書管理ではなく、**「本との関係」を管理する**ことを目的とする。

- 気になった本 / 読みたい本 / 読書中の本 / 読み終わった本

現在はまず「検索 → 気になる本保存」の体験をコアとして実装を進める。

**コアUX**: `本を見つける → 検索する → 気になる → 保存する → あとで見返す`

## 現在のMVP（実装対象）

### MVP-1. 本検索機能
**状態**: コードあり・仕様変更あり

**目的**: 本を探す。

| 項目 | 内容 |
|---|---|
| 検索方式 | キーワード検索（ボタン押下で実行、リアルタイム不要） |
| 使用API | Google Books API |

**検索結果表示項目**: 表紙画像 / タイトル / 著者 / 保存ボタン（☆ 気になる）

**変更点（仕様追加）**:
- 検索結果カードをタップ → 詳細画面へ遷移（現在は登録のみ）
- バーコード検索後も詳細画面へ遷移するように変更

---

### MVP-2. お気に入り保存機能
**状態**: コードあり・🐛 バグあり（保存ボタンを押しても保存済みにならない）

**目的**: 気になった本を保存する。

**保存時データ**: title / author / image_url / description / isbn

**保存思想**: 外部APIデータをそのまま参照し続けるのではなく、保存時に自分DBへ保存する。

---

### MVP-3. 保存済み状態表示
**状態**: コードあり・🐛 バグあり（検索時に保存済み表記が出ない）

**目的**: 重複保存防止・UX改善。

検索結果に対して「既に保存済みか」を表示する。（例: `✓ 保存済み`）

---

### MVP-4. お気に入り保存アニメーション
**状態**: 未実装

**目的**: 保存体験を気持ちよくする。

**イメージ**: `☆ → ★`

**優先度**: かなり高い。このアプリでは「保存する瞬間の体験」を重視する。

---

### MVP-5. 詳細画面拡充
**状態**: コードあり・機能追加

**目的**: 本の情報を充実させ、保存判断を助ける。

**追加表示項目**:
- あらすじ（現在あり）
- みんなの声・レビュー（外部レビュー連携）
- Amazon の商品ページ URL
- 楽天ブックスの商品ページ URL

## 重要思想

このアプリは「本検索サービス」ではなく、**「気になる本収集・管理サービス」** として設計する。

**重視するUX**:
- 保存しやすさ
- 保存済みの分かりやすさ
- 表紙による視認性
